# 🚲 共享单车用户行为分析 (Bike Sharing User Behavior Analysis)

> **团队项目** — 基于上海摩拜单车 2016 年 8 月数据的全链路分析
>
> 项目已按业务拆分为 **5 个 Notebook**，每个 Notebook 有明确的业务目标和输出，团队成员可认领分工。

---

## 📋 数据说明

### 骑行数据 (`上海共享单车数据.csv`)

| 字段 | 类型 | 说明 |
|------|------|------|
| `orderid` | int | 订单 ID（唯一） |
| `bikeid` | int | 单车 ID |
| `userid` | int | 用户 ID（匿名化） |
| `start_time` | datetime | 骑行开始时间 |
| `end_time` | datetime | 骑行结束时间 |
| `start_location_x` | float | 起点经度 |
| `start_location_y` | float | 起点纬度 |
| `end_location_x` | float | 终点经度 |
| `end_location_y` | float | 终点纬度 |
| `track` | string | GPS 轨迹点序列 |

**数据规模**: 102,361 条 · 16,887 名用户 · 79,062 辆单车 · 2016 年 8 月（上海）

### 天气数据 (`上海天气.xlsx`)

| 字段 | 说明 |
|------|------|
| 日期 | 日期 + 星期 |
| 最高温 | 含 ℃ 符号 |
| 最低温 | 含 ℃ 符号 |
| 天气 | 原始天气描述（**不简化**） |
| 风力风向 | 风力等级 |
| 日照时长 | 含"小时"单位 |

## 🎯 业务分析任务拆解

### 🅰️ Notebook A — 数据清洗与特征工程（✅ 已完成）

**业务目标**: 将原始脏数据清洗为可分析的数据，构造分析所需的基础特征。

| 步骤 | 具体内容 |
|------|---------|
| Step1 数据加载 | 加载骑行数据 + 天气数据，`info()` / `describe()` 查看全貌 |
| Step2 数据清洗 | 缺失值检查 → 时间类型转换 → 时长异常（<1min / >180min 剔除）→ 坐标异常（距离 >72km 剔除）→ 重复订单去重 |
| Step3 特征工程 | ① 骑行距离/速度/时长计算 ② 时间段分类 ③ 天气数据清洗（去℃去"小时"）④ 按日合并天气 ⑤ **按时间分档取温度（actual_temp）** ⑥ 用户级聚合 ⑦ 单车级聚合 |

**输出文件**: `共享单车_清洗后.csv`、`共享单车_用户特征.csv`、`共享单车_单车特征.csv`

**温度取值逻辑（已实现）**:

| 时段 | 小时 | 取值 |
|------|:----:|------|
| 🌙 夜间 | 20:00 ~ 7:00 | 最低温 |
| 🌤️ 上午 | 7:00 ~ 11:00 | 平均温 |
| ☀️ 下午 | 11:00 ~ 17:00 | 最高温 |
| 🌆 傍晚 | 17:00 ~ 20:00 | 平均温 |

### 🅲 Notebook C — 用户价值 RFM 分层（📢 待认领）

**业务目标**: 基于用户骑行行为，用 K-Means 聚类将用户分为不同价值等级，输出用户分层结果和运营建议。

| 步骤 | 业务角度 | 负责人 | 说明 |
|------|---------|:------:|------|
| **C1 加载用户特征** | 读取已有的用户聚合数据 | `____` | 使用 Notebook A 输出的 `共享单车_用户特征.csv` |
| **C2 R/F/M 定义** | 定义共享单车版的RFM | `____` | R=最近骑行天数, F=骑行次数, M=总里程 |
| **C3 数据标准化** | 消除量纲影响 | `____` | `StandardScaler` |
| **C4 手肘法定K值** | 确定最佳聚类数 | `____` | K=2~10 画簇内平方和图 |
| **C5 K-Means聚类** | 用户自动分群 | `____` | 选择最佳K值运行 |
| **C6 各簇特征分析** | 每个群有什么特点？ | `____` | R/F/M 均值对比表 |
| **C7 价值等级划分** | 定义 S/A/B/C 等级 | `____` | 按综合得分排序 |
| **C8 可视化展示** | 等级分布、RFM散点图 | `____` | 饼图、三维散点图 |
| **C9 运营建议** | 每类用户怎么运营？ | `____` | 差异化策略 |
| **C10 保存结果** | 输出分层数据 | `____` | `共享单车_RFM分层.csv` |

**用户价值等级说明**:

| 等级 | 特征 | 运营策略 |
|:----:|------|---------|
| **S (高价值)** | 最近常骑 + 频率高 + 里程长 | VIP待遇、月卡推荐、专属客服 |
| **A (中高价值)** | 较常使用、粘性中等 | 优惠券刺激、升级引导 |
| **B (中价值)** | 使用一般，有流失风险 | 推送提醒、分段优惠 |
| **C (低价值)** | 使用少或已流失 | 召回激励、新功能体验券 |

### 认领任务流程

```bash
# 1. 从 dev 拉最新代码
git checkout dev
git pull origin dev

# 2. 创建自己的功能分支
git checkout -b feat/eda-time

# 3. 写代码 → 提交 → 推送
git add .
git commit -m "feat: 添加各小时骑行量分布分析"
git push -u origin feat/eda-time

# 4. 在 GitHub 上提 Pull Request → 合并到 dev
```
